# M3U8/DASH 视频流录制可视化控制台 PRD

## 1. 项目概述

**项目名称**：Stream-Recorder Web UI (暂定)
**项目背景**：开源工具 `N_m3u8DL-RE` （https://github.com/nilaoda/N_m3u8DL-RE）提供了极其强大的流媒体（M3U8/DASH/ISM）解析与多线程下载混流能力。本项目旨在摒弃参考项目中传统或陈旧的系统界面，设计一个全新的、具有现代 SaaS 面板风格的 Web 可视化任务调度与监控系统。
**核心价值**：提供直观的任务看板、实时进度展示、便捷的批量操作以及完备的文件管理体验。

## 2. 技术栈架构规划

推荐采用全栈 TypeScript 方案构建前端和后端，搭配 Prisma 作为 ORM 工具，既能保证数据类型的安全，又利于前后端接口的对齐。整个系统可轻松容器化，后期非常适合打包成镜像直接部署在 1Panel 等 Docker 面板环境中进行集中管理。

* **前端**：Vue 3 或 React，搭配现代 UI 组件库 (如 Element Plus 或 Ant Design) 与 TailwindCSS 进行样式构建。
* **后端**：Node.js (如 Express 或 NestJS)，负责与底层录制进程交互、处理 API 请求。
* **核心引擎**：`N_m3u8DL-RE` 配合 `ffmpeg` (用于自动混流与格式转换)。
* **数据库**：SQLite。

## 3. UI/UX 布局重新设计思路

彻底打破旧版系统的堆叠式布局，采用左右分栏的现代后台管理界面：

* **全局导航 (Sidebar)**：左侧固定，提供“仪表盘”、“录制任务”、“视频库”、“系统设置”四大核心入口。
* **顶栏 (Header)**：展示全局状态 (系统 CPU/内存占用、当前下载总速) 以及明暗主题 (Dark/Light Mode) 切换功能。
* **主内容区 (Main Content)**：右侧宽屏展示，采用卡片 (Card) 和数据表格 (Data Table) 承载核心业务流。

## 4. 核心功能需求

### 4.1 仪表盘 (Dashboard)

提供服务器与录制任务的全局监控视角。

* **数据卡片**：展示“运行中任务”、“排队中任务”、“今日已完成”、“失败任务”的数量统计。
* **系统负载**：实时显示底层服务器的 CPU 占用率与内存使用率。
* **存储监控**：可视化展示目标存储目录的总容量与剩余空间，预防磁盘爆满导致录制崩溃。

### 4.2 任务管理 (Task Management)

系统的核心调度模块。

* **新建录制任务**：
* 支持单条添加与文本域批量导入多条 URL。
* 参数表单：流媒体 URL、自定义保存名称、选择保存子目录。
* 高级参数 (传递给底层引擎)：自定义 User-Agent、Headers、代理地址 (Proxy)、并发线程数、是否开启实时录制直播流。


* **任务队列列表**：
* 字段设计：任务名称、状态、实时速度、已下载大小、进度条、操作列。
* 实时交互：通过后端 WebSocket 推送，前端列表的进度条和速度实现平滑刷新，无需手动刷新网页。


* **任务控制操作**：
* 支持对单条或批量选中的任务进行：开始、暂停 (向底层引擎发送中止信号)、彻底删除、重试失败任务。



### 4.3 视频库管理 (File Management)

独立管理已录制完成的媒体文件。

* **视频列表**：以表格或网格模式展示已完成的 `.mp4` 或 `.mkv` 文件。包含文件名、大小、录制完成时间。
* **交互操作**：
* 浏览器内直接预览播放 (调用 HTML5 播放器)。
* 一键复制文件在服务器上的绝对路径。
* 删除视频源文件 (释放磁盘空间)。
* 下载视频至本地设备。



### 4.4 系统设置 (Settings)

持久化保存全局运行参数。

* **引擎配置**：设定 `N_m3u8DL-RE` 二进制文件与 `ffmpeg` 的绝对路径。
* **存储配置**：设定全局默认的视频保存根目录。
* **并发控制**：设置系统允许的最大同时运行任务数，超过该数值的任务自动进入排队状态。

## 5. 数据库表结构设计 (SQLite)

采用关系型结构，以下为使用 ORM 构建时的核心实体定义参考。

| 表名 (Entity) | 字段 (Fields) | 描述 / 数据类型 |
| --- | --- | --- |
| **Task** | `id` | 主键，UUID |
|  | `name` | 自定义视频文件名 (String) |
|  | `url` | 原始流媒体地址 (String) |
|  | `status` | 枚举：排队中、下载中、已完成、错误、已停止 (Int/Enum) |
|  | `progress` | 当前下载进度百分比 (Float) |
|  | `speed` | 当前下载速度 (String) |
|  | `process_id` | 记录运行该任务的底层系统 PID (Int，可空) |
|  | `created_at` | 任务创建时间 (DateTime) |
| **SystemSetting** | `key` | 配置项标识，主键 (String) |
|  | `value` | 配置内容，使用 JSON 字符串存储 (String) |

## 6. 核心技术难点提示

* **PID 进程追踪与安全停止**：Node.js 后端通过 `child_process.spawn` 启动 `N_m3u8DL-RE` 时，必须捕获并记录该进程的 PID。用户在页面点击“停止”时，需通过发送优雅中断信号 (如 `SIGINT`) 给该 PID，确保底层引擎能正常合并已下载的切片并输出有效视频文件，避免直接杀死进程产生损坏的碎片文件。
* **终端日志解析**：`N_m3u8DL-RE` 的进度、速度等数据是通过标准输出打印的终端字符。后端需要编写精确的正则表达式持续拦截 stdout/stderr 日志流，提取出百分比和速度数值，再通过 WebSocket 转发给前端。
